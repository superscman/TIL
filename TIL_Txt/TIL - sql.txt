2.오늘 새롭게 학습한 내용을 다른 사람에게 설명할 수 있나요?

데이터 베이스의 필요성
In-Memory는 프로그램의 실행에 의존함. 예기치 못한 상황에 데이터를 보호하지 못하고 프로그램이 종료되며 데이터를 받아올 수 없으며 데이터의 수명이 프로그램의 수명에 의존함.
File I/O는 데이터가 필요할 때마다 전체 파일을 매번 읽어야 해서 비효율적임. 파일이 손상되거나 여러 개의 파일을 동시에 다뤄야 할때 작업이 점점 힘들어짐.
관계형 데이터베이스는 하나의 CSV파일이나 엑셀 시트를 한 개의 테이블로 저장 가능.

SQL 소개
Structured Query Language(SQL)은 데이터 베이스 언어.
주로 관계형 데이터베이스 에서 사용(MySQL, Oracle, SQLite, PostgreSQL 등)
쿼리란 질의문, 검색할 때의 검색어, 데이터를 필터하기 위한 것.
SQL이란 데이터베이스 언어로 데이터베이스에 쿼리를 보내 원하는 데이터를 가져오거나 삽입 가능.
SQL을 사용하기 위해서는 데이터 구조가 고정되어 있어야 함.

SQL 기본 쿼리문
데이터베이스 생성: CREATE DATABASE 데이터베이스_이름;
데이터베이스 사용: USE 데이터베이스_이름;
테이블 생성: USE로 데이터베이스 선택 후 테이블 만들기 가능. 테이블은 필드와 함께 만들어야 함. CREATE TABLE 테이블_이름( 필드이름 필드타입 그외속성, 필드이름2 필드타입 그외속성, ...);
테이블 정보 확인: DESC 테이블_이름;
데이터셋에 포함될 특성 특정: SELECT 문자열, 숫자, 간단한연산, * 등
테이블과 관련한 작업을 할 경우: SELECT 특성1 FROM 테이블이름;
SELECT 특성1, 특성_2 FROM 테이블이름;
필터역할 쿼리문: WHERE 특성1 = '특성 값';
SELECT 특성_1, 특성_2 FROM 테이블이름 WHERE 특성_1 = '특성 값';

ACID
트랜잭션: 여러개의 작업을 하나로 묶은 실행 유닛
데이터베이스 트랜잭션 특성 ACID(Atomicity, Conisitency, isolation, Durability)
Atomicity(원자성): 전부 성공하거나 전부 실패하거나
Consistency(일관성): 하나의 트랜잭션 이전과 이후, 데이터베이스의 상태는 이전과 같이 유효해야 함.
isolation(격리성, 고립성): 모든 트랜잭션은 다른 트랜잭션으로부터 독립되어야 함.
Durability(지속성): 해당 기록은 영구적이어야 함.

SQL VS NoSQL
데이터 저장(storage)
NoSQL은 key-value, document, wide-column, graph등의 방식으로 저장
SQL을 이용해 데이터를 테이블에 저장. 미리 작성된 스키마 기반에 맞게 저장해야함.
스키마(schema)
NoSQL은 관계형 데이터베이스보다 동적으로 스키마 관리. 행을 추가할 때 새로운 열을 추가할 수 있음. 모든 열에 대한 데이터 반드시 입력하지 않아도 됨.
SQL은 고정된 형식의 스키마 필요. 데이터 속셩별로 열(column)에 대한 정보를 미리 정해두어야 함.
쿼리(Querying)
관계형데이터베이스는 테이블의 형식과 테이블간의 관계에 맞춰 데이터를 요청해야 함. SQL과 같은 구조화된 언어를 씀.
비관계형 데이터베이스의 쿼리는 데이터 그룹 자체를 조회하는 것에 초점을 둠. 구조화되지 않은 쿼리 언어로도 데이터 요청 가능
확장성(Scalability)
일반적으로 SQL기반의 관계형 데이터베이스는 수직적으로 확장. CPU사용 확장, 비용 높음. 매우 복잡. 시간많이걸림
NoSQL은 수평적 확장, 값쌈, 트래픽 편리하게 처리.
