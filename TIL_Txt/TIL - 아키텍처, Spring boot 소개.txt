2.오늘 새롭게 학습한 내용을 다른 사람에게 설명할 수 있나요?

아키텍처란?
건축 분야 유래 용어, 요구사항을 만족하는 건출물을 짓는 청사진, 컨셉을 잡고 최대한 심플하게 구성.
시스템 아키텍처: 하드웨어와 소프트웨어를 포함하는 어떤 시스템의 전체적인 구성을 큰그림으로 표현
소프트웨어 아키텍처: 하드웨어를 제외한 컴퓨터내의 모든 프로그램을 포괄하는 의미를 가지는 소프트웨어의 구성을 큰그림으로 표현
애플리케이션 아키텍처_계층형 아키텍처: API 계층(클라이언트의 요청을 받아들이는 계층, 표현계층), 서비스 계층(API계층에서 전달 받은 요청을 업무 도메인의 요구 사항에 맞게 비지니스적으로 처리하는 계층), 데이터 엑세스 계층(서비스계층에서 처리된 데이터를 데이터베이스 등의 데이터 저장소에 저장하는 계층)으로 나뉨.

아키텍처로 보는 Spring Framework 모듈 구성
모듈: Java에서는 일반적으로 지원되는 여러가지 기능들을 목적에 맞게 그룹화하여 묶어 놓은 것, 재사용 가능하도록 라이브러리 형태로 제공(대부분)
20여개의 모듈을 통해 다양한 기능 제공

Spring Boot란?
Spring Framework의 시작전에 해야하던 셋팅을 지원해주는 Spring Project중 하나
XML기반의 복잡한 설계 방식 지양
의존 라이브러리의 자동 관리
애플리케이션 설정의 자동 구성
프로덕션급 애플리케이션의 손쉬운 빌드
내장된 WAS를 통한 손쉬운 배포
Spring 구성은 Spring에게 맡겨버리고 비지니스 로직에만 집중하자!
